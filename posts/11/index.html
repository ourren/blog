PHP5.4中删除的安全函数

Mar 13th, 2012 12:00 am | 留言

前几天配置PHP5.4的时候发现PHP新版已经在一些安全函数方面进行了增改，从php自身防止一些不必要的漏洞，因此开发人员在写代码时也需要注意版本的兼容。下面讲几个涉及安全方面的函数进行说明。
1、safe_mode
状态：已删除
描述：此函数主要限制系统相关的文件打开、命令执行等函数(具体影响函数见参考1)。当然现在也有bypasss safe_mode的方法(见参考2)，通过com接口来绕过。
解决办法：可以通过配置php.ini中的disable_functions函数列表进行限制指定函数。

2、magic_quotes_gpc
状态：已删除 （magic_quotes_runtime、magic_quotes_sybase 删除）
描述：此函数在PHP <= 4.2.3默认开，会自动对GPC (Get/Post/Cookie)提交的值中的’, “, 和空格进行转义，PHP5.4中get_magic_quotes_gpc()默认返回false。
解决办法：基本上还是不受影响，先检测状态，然后采用mysql_real_escape_string进行转义处理。

3、register_globals
状态：已删除
描述：此函数虽然从PHP>>4.2开始默认由on改为off,但是如果程序员开启的话会导致变量覆盖等安全问题,特别是结合文件包含漏洞等。
解决办法：可以结合前面sablog源码中的函数进行改写(采用extract函数).

4、session_register
状态：已删除（session_unregister()、session_is_registered 删除）
描述：采用session认证的时候可以用该函数注册session变量。
解决办法：采用$_SESSION数组进行注册变量。

Nginx Php Mysql 安装与安全配置

Mar 11th, 2012 12:00 am | 留言

1、新建用户和权限设置

新建www用户组：

[code lang=“bash”]groupadd www -g 501[/code]

创建一个不能登陆、没有shell、没有主目录的用户：

[code lang=“bash”]useradd www -M -s /sbin/nologin -g 501[/code] 2、安装Pcre组件 [code lang=“bash”]# wget ftp://ftp.csx.cam.ac.uk/pub/software/programming/pcre/pcre-8.30.tar.gz

tar zxfv pcre-8.30.tar.gz

cd pcre-8.30

./configure

make && make install[/code]

3、修改、编译并安装nginx

修改banner信息

[code lang=“bash”]#src/core/nginx.h

define NGINX_VERSION “2.0.0”

define NGINX_VER “Apache/” NGINX_VERSION

define NGINX_VAR “Apache”[/code]

编译(最小化安装)

可以用命令查看现有配置

[code lang=“bash”]./configure –help |less[/code]

这里只用最简单的一些功能

[code lang=“bash”]./configure –prefix=/usr/local/nginx –user=www –group=www –with-http_stub_status_module –with-http_ssl_module make && make install[/code]

如果出现error while loading shared libraries: libpcre.so.1

解决办法(创建连接)：ln -s /usr/local/lib/libpcre.so.1 /lib

4、安装php

安装必要的组件 补充说明：PHP加密扩展库(libmcrypt)

[code lang=“bash”]yum -y install libjpeg-devel libpng-devel libxml2-devel curl-devel wget ftp://mcrypt.hellug.gr/pub/crypto/mcrypt/libmcrypt/libmcrypt-2.5.7.tar.gz tar -zxvf libmcrypt-2.5.7.tar.gz cd libmcrypt-2.5.7 ./configure make && make install[/code]

编译php并安装

[code lang=“bash”]wget http://us3.php.net/get/php-5.4.0.tar.gz/from/cn2.php.net/mirror tar zxfv php-5.4.0.tar.gz cd php-5.4.0 ./configure –prefix=/usr/local/php –with-iconv –with-zlib –enable-xml –disable-rpath –enable-safe-mode –enable-bcmath –enable-shmop –enable-sysvsem –enable-inline-optimization –with-curl –with-curlwrappers –enable-mbregex –enable-mbstring –with-mcrypt –with-gd –enable-gd-native-ttf –with-openssl –with-mhash –enable-pcntl –enable-sockets –with-xmlrpc –enable-zip –enable-soap –without-pear –with-mysql –with-mysqli –enable-sqlite-utf8 –with-pdo-mysql –enable-ftp –with-jpeg-dir –with-freetype-dir –with-png-dir –enable-fpm –with-fpm-user=www –with-fpm-group=www –enable-safe-mode make && make install[/code]

拷贝和修改php配置文件

[code lang=“bash”]cp php.ini-production /usr/local/php/lib/php.ini 或是 /usr/local/lib/php.ini cp /usr/local/php/etc/php-fpm.conf.default /usr/local/php/etc/php-fpm.conf /usr/local/php/bin/php –ini //测试ini文件是否加载[/code]

修改php.ini

[code lang=“php”]safe_mode = On //5.4已删除 register_globals = Off //5.4已删除 magic_quotes_gpc = Off //5.4已删除 allow_url_fopen = Off allow_url_include = Off expose_php=Off disable_functions = shell_exec,system,exec,passthru,show_source,curl_exec,curl_multi_exec,get_cfg_var [Date] date.timezone = “Asia/Shanghai”[/code]

修改php-fpm.conf

[code lang=“php”][global] pid = run/php-fpm.pid error_log = log/php-fpm.log log_level = notice emergency_restart_threshold = 0 emergency_restart_interval = 0 [www] pm.start_servers = 2 pm.min_spare_servers = 1 pm.max_spare_servers = 3 pm.max_requests = 500[/code]

添加服务启动脚本

[code lang=“bash”]vi /etc/init.d/nginx

!/bin/bash

chkconfig:345 60 60

description:nginx

/usr/local/nginx/sbin/nginx

vi /etc/init.d/php-fpm

! /bin/bash

chkconfig:345 61 61

description:php-fpm

/usr/local/php/sbin/php-fpm

chmod 755 /etc/init.d/nginx chmod 755 /etc/init.d/php-fpm

chkconfig –add nginx chkconfig –add php-fpm

chkconfig nginx on chkconfig php-fpm on[/code]

修改nginx配置文件

[code lang=“bash”]location ~ .php$ { root /home/wwwroot; fastcgi_pass 127.0.0.1:9000; fastcgi_index index.php; fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name; include fastcgi_params; }[/code]

在home/wwwroot/ 新建PHP文件测试成功。

5、权限设置

限制webshell访问其他目录

修改php.ini

[code lang=“bash”]open_basedir = “/home/wwwroot/”[/code]

补充:Apache 可以通过php_admin_value open_basedir “/tmp/:/usr/www/xpb/”进行限制

6、安装Mysql

创建mysql安装目录

[code lang=“bash”]mkdir -p /usr/local/mysql/[/code]

创建数据存放目录

[code lang=“bash”]mkdir -p /data/mysql/[/code]

创建用户和用户组与赋予数据存放目录权限

[code lang=“bash”]useradd mysql -M -s /sbin/nologin chown mysql.mysql -R /data/mysql/[/code]

安装必要的组件

[code lang=“bash”]yum -y install cmake yum -y install ncurses-devel[/code]

编译安装Mysql

[code lang=“bash”]cmake -DCMAKE_INSTALL_PREFIX=/usr/local/mysql -DMYSQL_UNIX_ADDR=/data/mysql/mysql.sock -DDEFAULT_CHARSET=utf8 -DDEFAULT_COLLATION=utf8_general_ci -DWITH_EXTRA_CHARSETS:STRING=utf8,gbk -DWITH_MYISAM_STORAGE_ENGINE=1 -DWITH_INNOBASE_STORAGE_ENGINE=1 -DWITH_MEMORY_STORAGE_ENGINE=1 -DWITH_READLINE=1 -DENABLED_LOCAL_INFILE=1 -DMYSQL_DATADIR=/data/mysql -DMYSQL_USER=mysql -DMYSQL_TCP_PORT=3306

make && make install[/code]

初始化数据库

[code lang=“bash”]cd /usr/local/mysql scripts/mysql_install_db –user=mysql –basedir=/usr/local/mysql –datadir=/data/mysql/[/code]

配置环境

[code lang=“bash”]cp support-files/my-medium.cnf /etc/my.cnf cp support-files/mysql.server /etc/init.d/mysql chmod 755 /etc/init.d/mysql chkconfig mysql on export PATH=/usr/local/mysql/bin:$PATH[/code]

启动并设置初始密码

[code lang=“bash”]/etc/init.d/mysql start mysqladmin -uroot password ‘123123’[/code]

进行再修改密码的语句

[code lang=“bash”]UPDATE mysql.user SET Password = PASSWORD(‘newpwd’) WHERE User = ‘root’;(生新设置密码) flush privileges;（刷新权限）[/code] 部分参考：http://www.4shell.org/archives/2095.html

Sablog源码笔记

Mar 8th, 2012 12:00 am | 留言

花了点时间分析了一下sablog2.0的安全和登陆过程，感觉自己原来写的函数确实考虑不周全，于是提取了几个比较实用的函数。

1、register_globals开启与否处理办法

解释：开启的话有可能导致变量覆盖问题，但是不会影响session认证等问题。如果未开启的话，可以考虑采用import_request_variables(‘G’)和extract($GET)进行类似处理，不过还是要注意处理自定义变量覆盖问题。 [code lang=“php”]// 允许程序在 register_globals = off 的环境下工作 $onoff = function_exists(‘ini_get’) ? ini_get(‘register_globals’) : get_cfg_var(‘register_globals’); if ($onoff != 1) { @extract($POST, EXTR_SKIP);//extract:将提交的数组键名作为变量名，值作为变量的值。 EXTR_SKIP：如果有冲突，不覆盖已有的变量。 @extract($_GET, EXTR_SKIP); }[/code] 2、转义问题

解释：如果GPC开始的话，我们使用addslashes后有可能造成双重转义，其实更好的函数是mysql_real_escape_string。 解决办法：如果GPC开启，先反转义，然后变量统一转义。 [code lang=“php”]// 去除转义字符 function sax_stripslashes(&$array) { if (is_array($array)) { foreach ($array as $k => $v) { $array[$k] = sax_stripslashes($v); } } else if (is_string($array)) { $array = stripslashes($array); } return $array; }[/code] [code lang=“php”]// 判断 magic_quotes_gpc 状态 if (get_magic_quotes_gpc()) { $GET = sax_stripslashes($GET); $POST = sax_stripslashes($POST); }[/code] 3、检测邮件名是否有效函数： [code lang=“php”]//判断是否为邮件地址 function isemail($email) { return strlen($email) > 6 && preg_match(“/^[w-.]+@[w-.]+(.w+)+$/”, $email); }[/code] 4、用户名敏感字符 [code lang=“php”]$name_key = array(“\”,‘&’,‘ ’,“‘”,’“‘,’/‘,’*‘,’,‘,’<‘,’>‘,”r”,“t”,“n”,’#‘,’$‘,’(‘,’)‘,’%‘,’@‘,’+‘,’?‘,’;‘,’^‘); foreach($name_key as $value){ if (strpos($username,$value) !== false){ $msg = “<p>用户名包含敏感字符.</p>”; $quit = TRUE; } }[/code] 5、跨站方面的防御

解释：任何用户的输入都可以造成危害，不管是GET、POST还是header，在页面输出的时候要进行编码。 [code lang=“php”]//防止一些低级的XSS if($SERVER[‘REQUEST_URI’]) { $temp = urldecode($SERVER[‘REQUEST_URI’]); if(strpos($temp, ‘<’) !== false || strpos($temp, ‘>’) !== false || strpos($temp, ‘(’) !== false || strpos($temp, ‘“’) !== false) { exit(‘Request Bad url’); } } //转换字符 function char_cv($string) { $string = htmlspecialchars(sax_addslashes($string)); return $string; }[/code] 6、IP地址相关 [code lang=”php”]//检测代理 $SERVER[‘HTTP_X_FORWARDED_FOR’] || $SERVER[‘HTTP_VIA’] || $SERVER[‘HTTP_PROXY_CONNECTION’] || $SERVER[‘HTTP_USER_AGENT_VIA’])

// 获得IP地址 if(getenv(‘HTTP_CLIENT_IP’) && strcasecmp(getenv(‘HTTP_CLIENT_IP’), ‘unknown’)) { $onlineip = getenv(‘HTTP_CLIENT_IP’); } elseif(getenv(‘HTTP_X_FORWARDED_FOR’) && strcasecmp(getenv(‘HTTP_X_FORWARDED_FOR’), ‘unknown’)) { $onlineip = getenv(‘HTTP_X_FORWARDED_FOR’); } elseif(getenv(‘REMOTE_ADDR’) && strcasecmp(getenv(‘REMOTE_ADDR’), ‘unknown’)) { $onlineip = getenv(‘REMOTE_ADDR’); } elseif(isset($SERVER[‘REMOTE_ADDR’]) && $SERVER[‘REMOTE_ADDR’] && strcasecmp($SERVER[‘REMOTE_ADDR’], ‘unknown’)) { $onlineip = $SERVER[‘REMOTE_ADDR’]; } $onlineip = char_cv($onlineip); @preg_match(“/[d.]{7,15}/”, $onlineip, $onlineipmatches); $onlineip = $onlineipmatches[0] ? $onlineipmatches[0] : ‘unknown’;[/code] 暂时就这么多，BTW，sablog采用的md5一次加密，认证成功后把加密密码放入cookies，感觉还是不妥当。
